## 1. Goal

Convert the repo from a flat Fumadocs app into a **bun workspace monorepo** so the
`@remocn/render-sdk` package can live next to the docs site. Move the existing docs app
into `apps/docs`; create an empty `packages/` slot for the SDK. No SDK code in this PR.

## 2. Why

The repo today is a single Next/Fumadocs app at the root (`app/`, `content/`, `lib/`,
`package.json` with `next build`). The SDK must be a separate publishable package in the
same repo (docs + library co-located, like files-sdk). A workspace is the prerequisite for
every later task ‚Äî it has zero product logic but unblocks all of them, so it ships first and
alone to keep the diff reviewable.

## 3. Scope

### In
- Root `package.json` becomes a **private workspace root**: `"workspaces": ["apps/*", "packages/*"]`,
  `"private": true`, no app `dependencies` (those move to `apps/docs`). Keep a thin set of
  root-level scripts that delegate (see ¬ß5).
- Move the entire docs app into `apps/docs/` **as a unit** (preserves all relative paths):
  `app/`, `content/`, `components/`, `lib/`, `source.config.ts`, `next.config.mjs`,
  `postcss.config.mjs`, `proxy.ts`, `tsconfig.json`, the current `package.json`, `README.md`,
  `app/global.css` (already under `app/`).
- `apps/docs/package.json`: `"name": "docs"`, `"private": true`, keep existing docs deps +
  scripts (`dev`/`build`/`start`/`types:check`/`postinstall`) unchanged.
- Create `packages/.gitkeep` (empty slot; the SDK lands in PR 02).
- Root `.gitignore` stays valid for the new layout (the existing globbed entries ‚Äî
  `node_modules`, `.next/`, `.source`, `next-env.d.ts`, `*.tsbuildinfo` ‚Äî already match by
  basename, so they keep working under `apps/docs`). Add `apps/docs/.next` only if needed.

### Out
- Any `packages/render-sdk` content (PR 02).
- CI, publishing, changesets.
- Touching docs *content* or components beyond the physical move.

## 5. Root scripts (delegating)

```jsonc
{
  "name": "remocn-render-sdk-monorepo",
  "private": true,
  "workspaces": ["apps/*", "packages/*"],
  "scripts": {
    "dev": "bun --filter docs dev",
    "build": "bun --filter '*' build",
    "docs:build": "bun --filter docs build"
  },
  "devDependencies": { "typescript": "^6.0.3" }
}
```
> `bun --filter` runs a script across matching workspaces. Exact script names confirmed
> during implementation against the installed bun version.

## 6. Acceptance criteria

- [ ] `bun install` at the root completes; a single hoisted root `node_modules` + `bun.lock`.
- [ ] `bun --filter docs build` (or `cd apps/docs && bun run build`) builds the docs site
      green ‚Äî no broken imports, `@/*` and `collections/*` path aliases resolve from the new
      location.
- [ ] `bun --filter docs dev` boots the docs site.
- [ ] Repo root contains **no** `app/`, `content/`, `next.config.mjs` etc. ‚Äî they live only
      under `apps/docs/`.
- [ ] `packages/` exists and is empty (`.gitkeep`).
- [ ] No reference to the old root paths remains (grep for `from "@/` resolves within `apps/docs`).

## 7. Test plan

- `bun install && bun --filter docs build` is the gate (no unit tests in this PR).
- Manual: open `http://localhost:3000` after `dev`, confirm docs render and search works.

## 8. Risks / notes

- The docs configs are **self-contained** (verified): `tsconfig` aliases are `@/* ‚Üí ./*` and
  `collections/* ‚Üí ./.source/*`; `source.config.ts` uses `content/docs`. Moving the directory
  as a unit keeps every relative path intact ‚Äî no path edits expected.
- `.source/` is generated (gitignored) and regenerated by `fumadocs-mdx` (`postinstall`); do
  not move it ‚Äî let it regenerate under `apps/docs`. Remove any stale root `.source/`.
- `bun.lock` will change (workspace resolution) ‚Äî expected; commit it.
- `.omc/` working dir stays at the repo root (not under any workspace).
