### Learnings / patterns for future PRDs
- bun isolated linker symlinks @types/bun into packages/render-sdk/node_modules/@types/bun,
  but TS auto-@types inclusion did NOT register its globals (Bun/process/node:*). Fix:
  explicit `"types": ["bun"]` in tsconfig.json — @types/bun pulls @types/node + bun-types.
- `bun test` with zero test files exits non-zero ("0 test files matching"). Ship at least one
  trivial test (or the build-output test) so the test gate is green.
- Importing `../src/index.ts` with the `.ts` extension fails tsc (TS5097) unless
  allowImportingTsExtensions; drop the extension — bundler resolution finds it.
- Bun.build `root: srcDir` is what preserves the dist/server, dist/lambda subdir layout so
  subpath export paths line up; without it everything flattens to dist/.
- Next: PR 03 (core types, errors, handle, codecs, SPI) writes into src/ — packaging frozen.
